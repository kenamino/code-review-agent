# 多智能体代码审查系统 - 项目描述

## 项目概述

我构建了一个**基于多智能体协作的代码审查与优化系统演示平台**（CodeReview AI），该系统展示了如何通过四个专业化的 AI Agent 进行分工合作，从代码结构、安全性、性能、文档四个维度提供代码分析。

## 系统架构与技术实现

### 多 Agent 协作框架

系统采用**四层 Agent 协作模型**，各 Agent 分工明确、并行执行：

| Agent | 职责 | 检测能力 |
|-------|------|--------|
| **代码分析 Agent** | 结构与风格分析 | 变量命名规范、代码复杂度、最佳实践违反 |
| **安全检测 Agent** | 安全漏洞识别 | XSS 攻击、SQL 注入、敏感数据泄露、动态代码执行 |
| **性能优化 Agent** | 性能瓶颈检测 | 算法复杂度、内存泄漏、不必要的网络请求、缓存策略 |
| **文档生成 Agent** | 文档质量评估 | 注释完整性、API 文档缺失、类型定义不清晰 |

### 技术栈

**前端：** React 19 + TypeScript + Tailwind CSS 4 + Framer Motion
- 代码编辑器界面（支持多种编程语言）
- 实时 Agent 执行流程可视化展示
- 结构化问题卡片组件（按严重等级分类：错误/警告/建议）
- 交互式统计仪表盘（Recharts 图表库）

**后端：** Express.js + tRPC + Drizzle ORM
- 类型安全的 RPC 接口（端到端类型推导）
- 异步 Agent 任务管理
- 审查结果持久化存储

**数据库：** MySQL/TiDB
- `review_tasks` - 审查任务记录
- `review_issues` - 问题检测结果
- `agent_logs` - Agent 执行日志
- `review_statistics` - 用户统计数据聚合

### 核心功能

#### 1. 代码提交与审查入口
- 支持代码粘贴或文件上传
- 支持 JavaScript、TypeScript、Python、Java、C++、C#、Go、Rust 等 8+ 种编程语言
- 一键触发多 Agent 协作审查流程

#### 2. 多 Agent 协作展示面板
- **可视化流程展示**：四个 Agent 的执行状态实时更新（待启动 → 运行中 → 已完成）
- **执行耗时统计**：每个 Agent 的平均处理时间与发现问题数
- **协作效果指标**：总问题数、严重等级分布

#### 3. 实时审查结果输出
- **结构化问题卡片**：每个问题包含标题、描述、严重等级、行号、修复建议
- **三级严重等级分类**：
  - **错误**：必须立即修复的严重问题
  - **警告**：应该处理的潜在问题
  - **建议**：代码质量改进建议
- **Agent 来源标签**：清晰标注每个问题由哪个 Agent 检测

#### 4. 项目统计仪表盘
- **关键指标**：总审查次数、发现问题总数、修复率
- **问题分布图表**：错误/警告/建议的比例分布（饼图）
- **问题分布柱状图**：各类型问题的数量展示

#### 5. 项目介绍与架构说明页
- 系统架构图与数据流展示
- Agent 角色详细介绍与能力说明
- 技术栈与集成方案说明

## 核心逻辑与实现细节

### Agent 协作流程

```
用户提交代码
    ↓
代码分析 Agent（并行执行）
    ↓
安全检测 Agent（并行执行）
    ↓
性能优化 Agent（并行执行）
    ↓
文档生成 Agent（并行执行）
    ↓
结果聚合 & 去重
    ↓
按严重等级排序
    ↓
返回结构化结果
```

### 问题检测示例

**代码示例：**
```javascript
var userInput = getUserInput();
element.innerHTML = userInput;
console.log("DEBUG: " + userInput);
```

**检测结果：**
| Agent | 问题 | 严重等级 | 建议 |
|-------|------|--------|------|
| 代码分析 Agent | 使用 var 而非 const/let | 警告 | 使用 const 声明不可变变量 |
| 安全检测 Agent | innerHTML 直接赋值用户输入（XSS 风险） | 错误 | 使用 textContent 或 DOM 方法 |
| 性能优化 Agent | console.log 在生产环境应移除 | 建议 | 使用日志库并在构建时移除 |
| 文档生成 Agent | 缺少函数注释与参数说明 | 建议 | 添加 JSDoc 注释 |

## 性能与规模

- **单次审查耗时**：平均 2-3 秒（四个 Agent 并行执行）
- **支持代码长度**：最长 10,000 行
- **支持的编程语言**：8 种（JavaScript、TypeScript、Python、Java、C++、C#、Go、Rust）

## 技术亮点

1. **多 Agent 并行执行**：四个 Agent 同时运行，充分利用计算资源
2. **智能去重与排序**：自动去除重复问题，按严重等级优先级排序
3. **类型安全的 API**：使用 tRPC 实现端到端的类型推导，消除运行时类型错误
4. **可扩展的 Agent 框架**：易于添加新的 Agent 或修改现有 Agent 的检测规则
5. **优雅的用户界面**：采用现代化设计（深色主题、流畅动画、响应式布局）
